## Calude Code VS Codex  VS  Kilo Code VS Qwen Code  VS CodeBuddy 

- 工具对比

    | 维度           | 🧠 Claude Code (Anthropic)                                    | ⚡ Codex (OpenAI)                              | 🔧 Kilo Code (开源社区)                        | ☁️ Qwen Code (阿里云)                                    | 🏢 CodeBuddy (腾讯)                                   |
    | :------------- | :----------------------------------------------------------- | :-------------------------------------------- | :-------------------------------------------- | :------------------------------------------------------ | :--------------------------------------------------- |
    | **一句话定位** | 模型原生的终端编程智能体                                     | ChatGPT内的编程智能体功能                     | 模型自由的开源CLI+IDE插件                     | 阿里云官方的全能AI编程代理                              | 深度集成腾讯生态的AI开发平台                         |
    | **交互形态**   | CLI为原生核心 + IDE插件                                      | CLI + IDE插件 + ChatGPT内置                   | CLI + IDE插件（VS Code/JetBrains）            | 独立CLI + IDE插件                                       | CLI + IDE + 插件 + 独立IDE                           |
    | **核心模型**   | Claude 全系 (Opus/Sonnet/Haiku)                              | GPT-5-Codex                                   | 500+模型 (支持Claude/GPT/Gemini/DeepSeek等)   | Qwen3-Coder 系列                                        | 腾讯混元 + 可接入DeepSeek等                          |
    | **定价模式**   | **订阅制** (含一定用量) + **API超量计费**                    | **订阅制** (含一定用量) + **API超量计费**     | **工具免费** + **API按量付费** (可选付费订阅) | **免费** (有一定配额) + **API按量付费** (可选付费订阅)  | **个人免费** + **订阅制** + **企业版**               |
    | **免费额度**   | API新用户：**$5** 试用金                                     | 免费版有**限额**；付费版包含一定用量          | 免费扩展 + **$20** 免费额度                   | 阿里云Coding Plan：个人免费版；新用户送**7000万**Tokens | 个人免费版：**250 credits/2周**；新用户：**7天**试用 |
    | **核心优势**   | Agent能力顶尖，全局上下文理解极强                            | 原厂集成，无缝融入ChatGPT生态，深度推理能力强 | 模型选择高度自由，成本透明，社区驱动          | 对Qwen模型优化充分，中文友好，与阿里云生态结合紧密      | 深度集成微信生态，企业级安全与协作功能强大           |
    | **主要局限**   | 价格昂贵，国内访问不便；需注意新出的第三方工具（如OpenClaw）使用费 | 功能相对单一，重度依赖OpenAI生态              | 上手有一定门槛，体验依赖于所选模型            | 对阿里云生态有依赖，国际化社区支持相对较弱              | 可能过于“自主”，存在失控风险，需要开发者监督         |

- Anthropic的Claude系列模型能力：
	- **Claude Opus 4.6**: 旗舰模型，专为复杂架构设计和长上下文理解而生，SWE-bench评分高达**80.8%**，是处理大型项目重构的“代码之神”。
	- **Claude Sonnet 4.6**: 中坚力量，在性能与成本间取得最佳平衡。拥有Opus 4.6约**90%** 的编码能力，但速度快3倍，价格更亲民。
	- **Claude Haiku 4.5**: 轻量级模型，主打极速响应和超低成本，适合高频、简单的编码任务。

- Qwen系列模型能力：
	- **Qwen3.6-Plus** 是阿里通义千问当前主推的编程智能体模型，被广泛视为国产编程能力的新标杆。它在代码生成、调试和自主Agent任务上相比前代有了大幅跃升，尤其在处理复杂项目和多步骤开发任务时表现突出，已经成为通义灵码等工具中“Auto模式”的默认首选模型。
	- **Qwen3.5-Omni** 是一款全模态模型，支持文本、图像、音频和视频输入。它的独特价值在于“Vibe Coding”——你可以通过口述需求或直接拍摄手绘草图、原型图，模型就能理解意图并生成对应代码，极大地降低了从想法到原型的时间成本。适合希望用更直觉、非结构化方式启动开发流程的场景。
	- **Qwen3-Coder** 是阿里专门为编程任务优化的开源模型系列，在开源社区中享有很高声誉。它的SWE-Bench评分达到69.6%，性能足以媲美多款闭源的顶尖模型，是构建自有编程智能体、需要本地部署或追求成本可控的开发者和团队的首选基座。
	- **Qwen3-Max** 是通义千问的旗舰级通用大模型，拥有万亿级参数规模，其综合能力在多个基准上超越了GPT-5等国际竞品。在编程方面，它的SWE-Bench评分与Qwen3-Coder持平（69.6分），证明了它即便作为通用模型，代码能力也已达到顶尖水平，适合需要同时处理复杂推理、知识问答和代码生成的高要求场景。
	- 关于**Qwen3.5-Max**，目前官方尚未公布其完整的技术细节和评测数据。作为“Max”系列的新成员，预计它将在Qwen3-Max的基础上进一步强化推理、编程和Agent能力，并有望在不久后正式发布，值得持续关注。
	- **Qwen2.5-Coder** 是上一代的开源编程模型，凭借扎实的性能和完全开源的特性，已成为众多衍生模型和社区项目的基石。例如SWE-Dev-32B等知名项目就是在它的基础上微调而来。对于希望深入研究模型架构或进行二次开发的场景，它仍然是一个非常可靠和稳定的选择。
